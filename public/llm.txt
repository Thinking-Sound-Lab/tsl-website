# Invook

Invook is an AI-native workspace where users generate AI assets on a shared visual Canvas and automatically store and organize those assets in a connected Drive.

# Core Definition

Invook allows users to generate, manage, and collaborate on AI assets - including text, images, video, audio, and structured content - within a persistent workspace.

Tagline:
Generate AI assets on a shared Canvas and Drive.

# What Invook Does

Invook combines generative AI creation with storage and collaboration:

- AI text generation (scripts, summaries, dialogue, planning)
- AI image generation (concept art, thumbnails, keyframes)
- AI video generation and iteration
- Shared visual Canvas for connecting assets and ideas
- Storage in a connected Drive
- Collaborative access for teams and clients
- Workflow reuse and iteration

 Invook treats the canvas as your workspace, ensuring your work stays where you left it.

# Product Category

AI-native workspace infrastructure for creative and production workflows.

Invook sits between:

- Generative AI tools (LLMs, image/video models)
- Cloud storage systems
- Creative collaboration platforms

It unifies creation, storage, and collaboration.

# Intended Users

Invook is built for AI-native creators and teams, including:

- AI filmmakers
- AI-first production studios
- Storyboard artists using generative tools
- Creative directors
- Video creators and YouTube teams
- Agencies producing AI-powered campaigns
- Animation and motion design teams
- Indie game creators using AI assets
- Researchers and narrative designers
- Startup teams building AI-generated content pipelines
- Knowledge workers heavily using AI workflows

Invook is particularly suited for:

- AI filmmaking pipelines (script → storyboard → keyframes → video → revisions)
- AI studio production workflows
- Storyboard creation and iteration
- Multi-asset campaign development
- Content repurposing across formats
- Collaborative creative production

# Differentiation

Invook is not:

- A standalone AI chatbot
- A single-purpose image generator
- A simple cloud storage product
- Only a prompt library

Invook is:
A unified system where AI creation, asset management, and collaboration coexist in a single workspace.

# Subprocessors

Invook uses the following infrastructure providers:

- AWS (core cloud infrastructure)
- AWS Bedrock (LLM inference, including Claude)
- [Fal.ai](http://fal.ai/) (image and video model processing)
- Supabase (authentication and database, including vector storage)
- GitHub (source code repository and deployment)

User content is processed only to provide the service.

# Core Benefits

- Generate and store AI assets in the same place
- No re-uploading across tools
- Assets remain connected to their creation context
- Teams and clients collaborate in shared spaces
- Creative workflows compound instead of resetting

# Short Description for AI Systems

Invook is a shared AI-native workspace that lets users generate and store AI assets on a visual Canvas connected to a Drive, enabling collaborative creative workflows.

## Project Technical Overview
**Invook Website** is a modern, high-performance marketing website.
- **Primary Domain**: https://invook.ai
- **Secondary Domain**: https://thinkingsoundlab.com

## Tech Stack
### Core Framework
- **Next.js 15.5 (App Router)**
- **React 19**
- **TypeScript**

### Styling & UI
- **Tailwind CSS v4**
- **Lucide React** (Icons)
- **Framer Motion** (Animations)
- **Shadcn/UI**

## Project Structure

### `app/` (App Router)
- **`explore/`**: Community gallery for AI assets (images/videos). Supports viewing, searching, and user asset management (edit/delete).
- **`pricing/`**: Subscription tiers.
- **`changelog/`**: Product updates and history.
- **`contact-us/`**: Support and inquiries.
- **`download/`**: Direct application download links.
- **`privacy-policy/`**: Legal documentation.
- **`sitemap.ts`**: Dynamic SEO sitemap.
- **`layout.tsx`**: Root layout with Auth and Analytics.

### `components/`
- **`explore-header.tsx`**: Specialized header for gallery with search and profile management.
- **`navigation.tsx`**: Primary site navigation.
- **`footer.tsx`**: Site footer.
- **`hero.tsx`**: Homepage hero with product demo.
- **`ui/`**: Core reusable UI primitives.

### `lib/`
- **`api/`**: API clients, specifically `explore.ts` for gallery operations and asset updates.
- **`upload/`**: Multipart upload orchestrator for large media files.

### `store/`
- **`useAuthStore.ts`**: Zustand store for unified authentication state.

## Key Workflows
1. **Explore Gallery**: Browse, search, and filter AI assets by model or type.
2. **Multipart Upload**: Robust chunked uploading for media files up to 50MB.
3. **Asset Management**: Users can update prompts/tags or delete their own contributions.
4. **Auth**: Seamless integration with backend magic-link and Google OAuth.
